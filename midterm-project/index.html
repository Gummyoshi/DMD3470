What is a CSS Animation?

CSS animations are a way to transition between styles, using one of two main methods, Keyframes or Transitions. Keyframes are used to make custom animations and transitions are usually for simpler animations between two states.

Transitions

The first way to animate with just CSS is the 'transition' keyword. This keyword carries four paramters with it, each serving a special purpose.

property

The property parameter is simple. It is the CSS property that you want to animate. Examples would be
color
background-color
transform
opacity
letter-spacing

duration

The duration parameter is, again, relatively simple. Here you specify how long you want the animation to take.

short (0.4s)

medium (1s)

long (3s)

timing-function

Timing-function controls how the speed of the animation changes over time. It determines whether the animation starts slowly, quickly, or moves at a constant rate. Examples include
ease
linear
ease-in
ease-out

delay

Delay simply controls how long the animation waits before it begins.
short (0.5s)
medium (1s)
long (2s)

Animations

The second way to animate with CSS is using the animation property together with @keyframes. These allow developers to create more complex and customizable animations. The animation property has eight parameters that control how an animation behaves.

animation-name

Animation-name specifies which @keyframes animation the element should use. The value must match the name of an existing keyframes block.

.box { animation-name: spin; }
@keyframes spin { }

animation-duration

Animation-duration determines how long one cycle of the animation takes to complete. It is measured in seconds or milliseconds.

animation-timing-function

Timing-function controls how the speed of the animation changes over time. It determines whether the animation starts slowly, ends slowly, or moves at a constant rate.

animation-delay

Animation-delay controls how long the animation waits before starting after it is triggered. This allows developers to stagger animations or synchronize them with other events.

animation-iteration-count

Animation-iteration-count determines how many times the animation will repeat. It can run a set number of times or continue forever.
animation-iteration-count: 3;

animation-iteration-count: infinite;

animation-direction

Animation-direction controls the direction the animation plays. It can play normally, in reverse, or alternate between forward and backward.
normal
reverse
alternate
alternate-reverse

animation-fill-mode

Animation-fill-mode determines which styles apply before the animation begins and after it finishes. This allows the element to keep the animation's final state.
none
forwards
backwards
both

animation-play-state

Animation-play-state controls whether the animation is currently running or paused. It is often used with hover interactions or JavaScript to start or stop animations.
animation-play-state: running;
animation-play-state: paused;

Keyframes

@keyframes in CSS defines the steps of an animation by specifying how styles change at different points during the animation. You set values at percentages (like 0%, 50%, 100%) or using from and to to describe the starting and ending states. The animation is then applied to an element using the animation property, which controls things like duration, timing, and repetition.

@keyframe example


@keyframes pulse{
 0% {
   transform: scale(1);
 }
 50% {
   transform: scale(1.4);
 }
 100% {
   transform: scale(1);
 }
}

Browser Support

modern browsers

The vast majority of modern browsers support CSS animations. If they don't, make sure to have your non-animated state also look good. This way if the animation is unable to play, it doesn't look awful.

Applications

practical usage

CSS animations are often used for buttons, making them change color or slightly grow when clicked so users know their action worked. They're also used for loading indicators and progress animations, which show that a website is processing something instead of being frozen. On top of this, CSS animations help create smooth transitions and engaging interfaces, like elements fading in when scrolling or menus sliding open. Libraries like AOS use JavaScript to funciton, so they weren't touched on during this project.
